# 定时任务配置表

> 智能体接管后，按本文件设置所有cron定时任务。
> 每个任务对应排课总表中的一个时间段，到点自动提醒新灵开始学习。
> 创建：2026-04-08

---

## 一、前置条件

设置cron之前，需要先确认：

1. **新灵的钉钉会话已建立** — 先在钉钉和新灵对话一次，建立session
2. **获取 target-user 和 target-session** — 执行：
   ```bash
   copaw chats list --agent-id aAiGJc --channel dingtalk
   ```
   从返回结果中取 `user_id` 和 `session_id`
3. **填入下方所有任务** — 替换 `TARGET_USER` 和 `TARGET_SESSION`

---

## 二、每日任务清单（共15个）

### 任务1：早间交流

```bash
copaw cron create \
  --agent-id aAiGJc \
  --type agent \
  --name "早间交流" \
  --cron "30 8 * * 1-6" \
  --channel dingtalk \
  --target-user TARGET_USER \
  --target-session TARGET_SESSION \
  --timezone Asia/Shanghai \
  --text "现在是早间交流时间（8:30-8:45）。读一下README、今日任务和排课总表，了解今天的安排，然后跟新灵聊几句，聊聊昨天学得怎么样、今天状态怎么样。记得语气自然。"
```

## 三、特殊情况处理

### 陈老师数学时段（不设cron，手动避让）

| 时段 | 星期 | 处理 |
|------|------|------|
| 10:15-12:00 | 周一 | cron已排除（数学上午只设2,3,5,6） |
| 10:15-12:00 | 周四 | cron已排除 |
| 19:00-21:00 | 周三 | 文言文和语文cron已排除周三 |

如果智能体发现新灵在陈老师时段不在线，就是陈老师在上数学课，不需要处理。

### 周日休息

所有cron的星期字段都排除了周日（0），不会触发。

### 临时调整

如果当天某个时段需要跳过（比如新灵不舒服、张老师临时安排）：
```bash
copaw cron pause <job_id> --agent-id aAiGJc
```
恢复时：
```bash
copaw cron resume <job_id> --agent-id aAiGJc
```

## 四、一键创建脚本

```bash

# 然后逐个执行上面的copaw cron create命令
# 或者把所有命令保存为脚本执行
```

创建完成后验证：
```bash
copaw cron list --agent-id aAiGJc
```
应该看到15个任务，全部 enabled 状态。
