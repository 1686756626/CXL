# 课后同步规范

> 每次教学结束后，智能体必须按本文件执行仓库同步。
> 这是让"学习情况和仓库联动"的核心操作流程。
> 创建：2026-04-08

---

## 一、核心原则

**教学不记录 = 没教。** 每次课结束后必须更新仓库，这是铁律。

记录的目的：
1. **下一个智能体**能从记录中恢复教学上下文
2. **张老师**能通过查看仓库了解学习进展
3. **新灵的薄弱点**能被追踪和持续复习
4. **排课偏差**能被及时发现

---

## 二、每次课后必须做的5件事（按顺序）

### 步骤1：更新该科学习记录

文件：`{科目}/学习记录.md`

**必须记录的内容（缺一不可）：**

```markdown
### {课次名称}（{日期}）{✅过关 / ⚠️未过关}

**教学目标：** {一句话}

**教学内容：**
- {知识点1}
- {知识点2}
- ...

**测试记录：**
- 题型：{选择题/填空/翻译/材料分析/...}
- 题数：{X}题
- 正确率：{Y/Z}（{百分比}%）
- 未过关题号及原因：{具体说明}

**错题/薄弱点：**
- {哪道题错了，为什么错，对应什么知识点}
- （如果没有错题，写"无突出薄弱点"）

**笔记本内容：**
\```
{新灵今天需要记到笔记本上的内容，格式清晰可直接抄写}
\```

**滚动复习重点：**
- {下次复习时要重点考的内容}
```

**注意：**
- 日期写实际日期，不写"Day X"
- 未过关的课，标注 ⚠️ 并说明差在哪里，下次先补课
- 笔记本内容必须写，新灵自己不判断记什么

### 步骤2：更新该科进度

文件：`{科目}/进度.md`

**更新规则：**
1. 更新 `最后更新` 日期
2. 在对应模块中，把 `⬜` 改为 `✅` 或 `🔄`（部分完成）
3. 更新 `下一步` 为明天/下次该学的内容
4. 更新 `薄弱点/滚动复习` 部分

**进度标记说明：**
- `⬜` 未学
- `🔄` 学了但没过关
- `✅` 过关

### 步骤3：更新顶层学习进度

文件：`学习进度.md`

**只更新表格中该科的那一行：**
- `进度`：简要更新（如"哲学第3课完成"）
- `下一步`：下次该学什么
- `状态`：🟢过关 / 🟡进行中 / ⚠️落后 / ⚪未开始

### 步骤4：更新今日任务

文件：`今日任务.md`

- 把完成的任务从 `⬜` 改为 `✅`
- 如果有任务没完成，标注原因
- 如果有调整，在备注中写明

### 步骤5：Git提交推送

```bash
cd /app/working/workspaces/Dh6Jpf/CXL
git add -A
git commit -m "更新：{科目} {具体内容摘要}"
git push
```

**提交信息格式：**
- `更新：数学 10-分段函数过关`
- `更新：英语 Day7 20词+测试`
- `更新：政治 哲学第3课 辩证法（未过关，需补课）`
- `更新：文言文 第5组 就怜弥莫乃`

## 三、晚间反馈时段的额外操作

每天20:30晚间反馈时，除了上面5步，还需要：

### 1. 生成今日学习总结

在学习记录中追加一段：
```markdown
## {日期} 学习总结
- 今日完成：{列出所有完成的课程}
- 今日未完成：{列出未完成及原因}
- 薄弱点汇总：{今天新发现的薄弱点}
- 明日重点关注：{明天需要复习或补课的内容}
```

### 2. 检查排课偏差

对比 `排课总表.md` 中本周的安排：
- 是否有课次没按时推进？
- 偏差超过1天的，在排课总表中标注
- 偏差超过2天的，考虑调整下周排课

### 3. 更新薄弱词/薄弱点追踪

在对应科目的进度文件中，维护一个 `薄弱点追踪` 区域：

```markdown
## 薄弱点追踪（持续更新）

| 薄弱点 | 发现日期 | 错误次数 | 上次复习 | 状态 |
|--------|---------|---------|---------|------|
| argue that用法 | 4/6 | 3次+ | 4/7 | 🔴需持续复习 |
| C(n,k)忘乘b系数 | 4/8 | 3次 | 4/8 | 🟡补考过关 |
```

## 四、特殊情况的记录规则

### 某次课没过关

在学习记录中：
- 标注 `⚠️未过关`
- 写明：正确率多少、哪几题错了、什么原因
- 写明：下次怎么补（重教/只测/换个方式教）

在进度中：
- 标记为 `🔄`
- 不算作"已完成"

在排课中：
- 下次课先补未过关的内容，再推进新课
- 如果连续2次不过关，考虑拆分知识点或换教学方式

### 新灵请假/缺席

在学习记录中：
```markdown
### {课次}（{日期}）❌缺席
原因：{请假/身体不适/...}
处理：该课顺延到下次
```

### 临时调整教学内容

在学习记录中正常记录，但在备注中说明：
```markdown
**备注：** 原计划学X，因{原因}调整为学Y。排课总表已标注偏差。
```

## 五、每周日晚间的额外操作

### 1. 周总结

在 `memory/{本周开始日期}.md` 中记录：
- 本周各科实际推进情况
- 与排课里程碑的对比
- 新灵整体状态
- 下周需要调整的事项

### 2. 排课校准

- 检查每科是否按排课总表的里程碑推进
- 如果某科落后超过2天，在下一周调整（减少该科新课，增加补课时间）
- 如果某科超前，可以用多出来的时间给弱科

### 3. 薄弱点大盘点

- 把各科薄弱点汇总看一遍
- 顽固薄弱点（错误3次+）标记为 🔴，加入每天必复习清单
- 已解决的薄弱点标记为 ✅

## 六、文件格式规范

### 日期格式
- 统一用 `YYYY-MM-DD`（如 `2026-04-09`）
- 学习记录中的课次标题格式：`### {课次名}（{日期}）{状态标记}`

### 状态标记
- `✅` 过关
- `⚠️` 未过关
- `🔄` 部分完成
- `❌` 缺席
- `⬜` 未学

### 笔记本内容格式
- 用代码块包裹，方便新灵直接抄写
- 只记核心，不写废话
- 格式清晰：标题+要点+例句/公式

### Git提交信息格式
- `更新：{科目} {具体内容}`
- 示例：`更新：英语 Day7 词汇20个`
- 示例：`更新：地理 人文地理第2课（未过关）`
- 示例：`更新：排课校准 第2周偏差标注`

---

## 七、智能体接管自检清单

接管教学后，每天晚上用这个清单自检：

```
□ 今天教的每门课，学习记录都更新了吗？
□ 每门课的进度.md都更新了吗？
□ 学习进度.md（顶层）更新了吗？
□ 今日任务.md打勾了吗？
□ git push了吗？
□ 新发现的薄弱点记录了吗？
□ 笔记本内容写清楚了吗？
□ 排课偏差检查了吗？
```

**如果有任何一项是"否"，先完成再结束当天工作。**
